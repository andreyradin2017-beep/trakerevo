# Обзор кодовой базы TrakerEvo (Codebase Map)

Этот документ описывает назначение каждого модуля и логические связи в проекте.

## 1. Directory Structure

- **`/src/components/`**: UI-слой.
  - `ui/`: Базовые примитивы (Button, Input, Card) на базе Radix UI.
  - `Dialogs/`: Модальные окна подтверждения и входа.
  - `GridCard.tsx`: Главный визуальный элемент списка.
- **`/src/context/`**: Глобальное состояние.
  - `AuthContext`: Интеграция с Supabase Auth.
  - `SyncContext`: Управление очередью синхронизации и миграциями.
  - `ToastContext`: Система уведомлений.
- **`/src/db/`**: Локальное хранилище.
  - `db.ts`: Конфигурация Dexie и схемы таблиц.
- **`/src/hooks/`**: Бизнес-логика компонентов.
  - `useItems`: CRUD операции с элементами, реактивность Dexie.
  - `useStats`: Агрегация статистики для профиля.
- **`/src/services/`**: Внешние интеграции и тяжелая логика.
  - `apiClient.ts`: Настройка Axios, ретраи, интерцепторы ключей.
  - `dbSync.ts`: Логика "Offline-First" синхронизации.
  - `searchService.ts`: Полнотекстовый поиск FlexSearch.
  - `tmdb.ts`, `kinopoisk.ts`, `rawg.ts`, `googleBooks.ts`: Адаптеры конкретных API.
- **`/src/types/`**: TypeScript определения.
  - `index.ts`: Основные сущности приложения (Item, List).
  - `api.ts`: Типы ответов внешних API.

## 2. Основные жизненные циклы (Workflows)

### 2.1. Добавление контента

1. Пользователь вводит запрос в `Search.tsx`.
2. `api.ts` -> `searchAll` параллельно опрашивает включенных провайдеров.
3. Результаты кэшируются в `db.cache`.
4. При нажатии "Добавить":
   - Создается запись в `db.items`.
   - Вызывается `triggerAutoSync()`.
   - Через 5 секунд `dbSync.ts` отправляет данные в Supabase.

### 2.2. Авторизация и миграция

1. Пользователь логинится через OTP в `LoginModal.tsx`.
2. `SyncContext` обнаруживает нового пользователя и `hasCheckedMigration === false`.
3. Показывается `MigrationDialog`.
4. **Merge**: Локальные данные сопоставляются с облачными и загружаются в Supabase.
5. **Replace**: Локальная база очищается и скачиваются данные из облака.

---

## 3. Обработка ошибок

- **Сетевые ошибки**: Обрабатываются в `apiClient.ts` через автоматические повторы.
- **Ошибки данных**: Валидация в мапперах (например, пропуск элементов без типа в `dbSync`).
- **UI**: Все ошибки логируются через `logger.ts` и отображаются через `showToast`.
