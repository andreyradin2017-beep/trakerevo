# Техническая спецификация TrakerEvo: Детальный разбор

## 1. Архитектурная философия

TrakerEvo построен на принципе **Offline-First Resilience**. Это означает, что приложение должно работать идеально даже при полном отсутствии интернета, используя локальную БД как единственный источник правды для UI.

---

## 2. Слой данных (Data Layer)

### 2.1. Dexie.js (IndexedDB)

Используется версия схемы **12**. Основные таблицы:

- `items`:
  - _Индексы_: `++id`, `type`, `status`, `isArchived`, `*tags` (мульти-индекс), `listId`, `createdAt`, `externalId`, `source`, `[externalId+source]` (составной ключ), `supabaseId`.
- `search_history`: Хранит последние 50 запросов для быстрых подсказок.
- `cache`: TTL-кэш для ответов API (по умолчанию 30 дней).

### 2.2. Синхронизация (Cloud Sync)

Реализована в `services/dbSync.ts` и `context/SyncContext.tsx`.

- **Механизм**:
  - `toRemoteItem` / `toLocalItem`: Мапперы для нормализации данных между Dexie (CamelCase) и Supabase (Snake_case).
  - `syncDeletions`: Очередь удалений. Если предмет удален в офлайне, его ID попадает в `deleted_metadata`. При синхронизации выполняется `DELETE` в Supabase.
  - **Deduplication**: При миграции гостевых данных используется поиск по `externalId` + `source`.
  - **Data Consistency**: Функция `ensureDataConsistency` (в `dbSync.ts`) консолидирует задачи обслуживания (миграция статусов, очистка кэша, нормализация) в рамках единой транзакции перед синхронизацией.

---

## 3. Сервисы API (The Infinity Engine)

### 3.1. Нормализация данных (`services/normalizer.ts`)

Централизованный сервис для стандартизации данных из различных источников (TMDB, RAWG, Kinopoisk, OpenLibrary).

- **Функции**:
  - `normalizeItem`: Исправление типов, статусов (например, `dropped` -> `planned`) и путей к изображениям.
  - `fixLegacyImageUrl`: Автоматическое перенаправление устаревших прокси-ссылок на `wsrv.nl`.
- **Интеграция**: Используется в `kinopoisk.ts`, `dbSync.ts` и при сохранении новых элементов.

### 3.2. Агрегатор (`services/api.ts`)

Центральный узел, который распределяет запросы между провайдерами:

- **SearchAll**: Параллельный запуск задач через `Promise.allSettled`.
- **Smart Category Search**: Автоматический выбор провайдера на основе категории (например, для игр — только RAWG).

---

## 4. Поиск и Индексация

**FlexSearch** (`services/searchService.ts`) используется для сверхбыстрого локального поиска.

- **Тип индекса**: `Document` (позволяет искать по нескольким полям сразу).
- **Авто-индексация**: Реализована через **Dexie Hooks** (`creating`, `updating`, `deleting`). Любое изменение в таблице `items` автоматически синхронизируется с индексом FlexSearch без ручного вызова `upsertItem`.
- **Конфигурация**: `tokenize: "full"` — позволяет находить совпадения в середине слова.
- **Области**: `title`, `description`, `tags`, `original_title`.

## 6. Инструкции по проверке (Verification)

Для проверки любой новой функциональности:

1. Запустить `npm run test` для проверки регрессий в API и логике синхронизации.
2. Проверить `Console -> Network`, чтобы убедиться, что ретраи работают.
3. Проверить `src/test/normalizer.test.ts` для верификации логики нормализации.
