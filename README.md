## 🚀 Архитектурные особенности

### 1. Гибридная синхронизация (Hybrid Cloud)

Приложение работает по принципу **Optimistic UI**. Изменения применяются в локальную базу мгновенно, а `dbSync` сервис в фоновом режиме отправляет их в Supabase. При входе через аккаунт срабатывает алгоритм слияния данных (Merge), предотвращающий дубликаты по `externalId`.

### 2. Мульти-провайдерный поиск

Единый интерфейс поиска агрегирует данные из 4-х источников:

- **TMDB**: Фильмы и Сериалы.
- **RAWG**: Видеоигры.
- **Google Books / LitRes / Bookmate**: Книги.
- **Kinopoisk**: Локализованные данные для RU-сегмента.

### 3. Bento UI & Gamification

Статистика и профиль пользователя выполнены в премиальном стиле Apple Bento Grid. Система опыта (XP) за добавление и завершение контента геймифицирует процесс ведения архива.

## 📂 Структура проекта

- `src/services/api.ts` — Агрегатор внешних запросов и кэширования.
- `src/db/db.ts` — Схема локальной БД (Dexie).
- `src/pages/ItemDetail.tsx` — Центральный компонент просмотра контента с динамическими метаданными.
- `src/utils/haptics.ts` — Мост для тактильной отдачи (Web + Telegram).
- `src/test/` — Комплексное покрытие тестами (Animations, Discover, API).
